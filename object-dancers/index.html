What is the benefit of your class not relying on any code outside of its own definition?

The benefit of not relying on outside code is that each code can have its own variable. This means you could have many different drawings and functions working together without any worry of overlap. This can be especially helpful for group projects or if you are creating a very long and complicated code.


What makes it challenging to write code that has to harmonize with the code other people have written?

The challenge is not having code that overlaps. If there is an identical function or variable, then the code might cause an error. So, it is important not to use global variables. Additionally, it is important to keep labeling each code and using console.log. This makes it easier to track if an issue ever arises.
